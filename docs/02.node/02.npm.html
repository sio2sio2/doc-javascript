2.2. Paquetes¶

Hay muchas librerías y aplicaciones que se distribuyen en paquetes npm y que podemos integrar en nuestros propios desarrollos. Por otro lado, también podemos contribuir a la comunidad aportando nuestros propios desarrollos.

2.2.1. Gestión¶

2.2.1.1. Instalación¶

Antes de proceder a la instalación de cualquier paquete es necesario saber que estos se pueden instalar:

  • Globalmente, para que estén disponibles en todo nuestros desarrollos.

  • Localmente al directorio de trabajo de la aplicación que estamos desarrollando.

Para una instalación global, desde cualquier lugar del sistema de ficheros podemos hacer:

# npm install -g [nombre-paquete]

La instalación es local, en cambio, requiere situarnos en el directorio de trabajo y prescindir de la opción -g. Además, al instlar podemos distinguir entre dos clases de paquetes distintos según sea la dependencia que de ellos tenga nuestro desarrollo.:

  • Paquetes cuya función es permitirnos desarrollar y generar la aplicación. Por ejemplo, si desarrollamos una aplicación de cliente, necesitaremos webpack para darle forma de aplicación de cliente.

    Para instalar paquetes de desarrollo, es necesario añadir la opción --save-dev:

    $ npm install --save-dev webpack

  • Paquetes necesarios para que la aplicación desarrollada se ejecute. Por ejemplo, si desarrollamos una aplicación GIS, podemos basarnos en la libreria Leaflet.

    En este caso es necesario añadir la opción --save (o nada en absoluto):

    $ npm install --save leaflet

En ambos casos, las instalaciones se registran en el fichero package.json, lo cual posibilita que si se obtiene un directorio de una tercera persona (por ejemplo, porque haya subido el desarrollo a Github y nosotros lo hayamos clonado), podemos instalar todas las dependencias simplemente haciendo:

$ npm install

La instalación global tiene la ventaja de que permite compartir una misma instalación entre todos los proyectos que desarrollemos. En cambio, no deja registro en package.json; y, en consecuencia, dificulta la reproducibilidad del proyecto, esto es, que un tercero pueda descargar el repositorio del proyecto y, sin demasiadas preocupaciones, pueda instalar el software necesario.

Nota

Es también posible instalar directamente un repositorio de Github si éste está construido apropiadamente (tiene un fichero package.json):

$ npm install git+https://github.com/usuario/proyecto.git

2.2.1.2. Búsqueda¶

Podemos buscar paquetes instalables con npm search:

$ npm search webpack

Y paquetes instalados con npm list:

$ npm list -g --depth=0
$ npm list --depth=0

La primera orden busca exclusivamente paquetes instalados globalmente. Además, incluyendo --depth=0 no se listan dependencias.

2.2.1.3. Actualización¶

Para actualizar un paquete concreto, puede usarse:

$ npm update paquete

y para actualizar todos, basta con prescindir de cualquier nombre:

$ npm update

2.2.1.4. Desinstalación¶

Puede usar npm uninstall (o su sinónimo npm remove):

$ npm uninstall leaflet

2.2.2. Publicación¶

Si lo consideramos oportuno, también es posible publicar nuestros propios desarrollos en los repositorios de npm. Lo indispensable es tener guardado el desarrollo en un directorio de trabajo (o sea, que disponga de un fichero package.json). Es especialmente importante que en package.json estén perfectamente definidos:

  • name, con el nombre de la aplicación.

  • version, con el número de versión apropiado.

  • description, con la descripción.

  • keywords, con una lista de palabras clave.

La importancia estriba en que npm lee este fichero para extraer los datos de la aplicación, por lo que exclusivamente esos valores serán los que permitan al servicio de paquetería identificar y clasificar al software.

2.2.2.1. Registro¶

Lo primero es registrar nuestro propio usuario en la página oficial, para lo cual nos requerirán un nombre de usuario, una contraseña y un correo electrónico.

2.2.2.2. Subida¶

La subida del paquete es sumamente sencilla. Basta con situarse en el directorio de trabajo y:

  1. Ingreso:

    $ npm login

    El ingreso nos proporcionará un token:

    $ npm token list

    que servirá como método de autenticación al publicar el paquete.

  2. Publicación del paquete:

    $ npm publish

  3. Opcionalmente, podemos deslogearnos:

    $ npm logout
